소개

안녕하세요!. 웹스토리보이입니다. 오늘부터 Three.js를 공부해보겠습니다. Three.js를 작업하기 전에 먼저 설치부터 작업하겠습니다. Three.js를 작동하는 방법은 여러가지를 제공합니다. 간단한 작업들은 다운을 받거나 CDN을 이용하는 것이 좋으며, 프로젝트 규모가 커지거나 소스가 많아진다면 webpack이나 npm을 이용하는 것이 좋습니다. 여러분들 상황에 맞게 적절한 방법을 선택하여 적용하시면 됩니다.
1. 파일을 다운 받아서 작업하는 방법
2. CDN을 통해서 작업하는 방법
3. NPM을 이용해서 작업하는 방법
4. webPack을 이용해서 작업하는 방법
5. vite를 이용해서 작업하는 방법

1. 파일을 다운 받아서 작업하는 방법

파일은 three.js 사이트에서 다운 받으면 된다. build 폴더에 three.js 관련 파일들이 있고, 연동하고 확인해보면 Three.js 객체를 불러오는 것을 확인 할 수 있습니다. examples/jsm 폴더에는 여러가지 설정 파일들도 들어있어, 필요한 것은 연동하여 사용하면 됩니다. 

<!DOCTYPE html>
    <html lang="ko">
    <head>
        <meta charset="UTF-8">
        <meta http-equiv="X-UA-Compatible" content="IE=edge">
        <meta name="viewport" content="width=device-width, initial-scale=1.0">
        <title>Three.js</title>
    </head>
    <body>
        <script src="build/three.min.js"></script>
        <script>
            console.log(THREE)
        </script>
    </body>
    </html>

2. CDN을 통해서 작업하는 방법

다운 받기 귀찮다면 CDN을 통해서 작업할 수 있습니다. CDN은 검색을 통해서 적용할 수 있으며, 개인 서버에 올려서 작업이 가능합니다. 

<script src="https://cdnjs.cloudflare.com/ajax/libs/three.js/r127/three.min.js">

3. NPM을 이용해서 작업하는 방법

node.js를 설치하고 NPM을 이용하여 설정하는 방법입니다. 터미널을 이용해서 작업하기 때문에 설치가 쉽습니다. 터미널을 열고 다음과 같이 작성하면 파일들이 자동으로 생성됩니다. node_modules 폴더가 생기고 build, examples, src 폴더가 생성이 되며, 파일을 만들고 작업하면 됩니다. 

npm install --save three

4. webPack을 이용해서 작업하는 방법

webpack은 HTML, CSS, javascript 등의 파일을 하나로 압축하여 만들어주는 컴파일러입니다. 처음에 사용하기 위해서는 설정해야 할 부분들이 많고, 복자하기 때문에 보통 누군가가 잘 만들어 놓은 webpack을 사용하면 쓰기가 수월합니다. https://github.com/aakatev/three-js-webpack 

npm init

npm을 초기화하기 위한 명령어입니다. 초기 셋팅에 필요한 정보를 설정하는 부분입니다.

npm install -D webpack webpack-cli

webpack을 인스톨하기 위해 webpack과 cli를 설치합니다. node_modules 폴더가 생기고 다음과 같이 파일이 생성됩니다. 

"devDependencies": {
    "webpack": "^5.74.0",
    "webpack-cli": "^4.10.0"
}

webpack.config.js 파일을 만들고 다음과 같이 셋팅을 합니다. 

const path = require("path");

module.exports = {
    mode: "development",    // 개발 모드를 설정하고
    entry: {                // 엔트리 파일을 설정하고
        main: "./src/app.js",
    },
    output: {                               // 아웃풋 파일을 만듭니다.
        path: path.resolve("./dist"),
        filename: "[name].js",
    },
};

app.js 파일을 만들고 다음과 같이 셋팅을 합니다. 

import { view } from "./main.js";

window.addEventListener("DOMContentLoaded", () => {
    view();
});

main.js 파일을 만들고 다음과 같이 셋팅을 합니다. 

export function view() {
    console.log("hello world");
}

이렇게 하고 npm run build를 하면 dist 폴더가 생성되고 파일이 하나로 합쳐진 것을 볼 수 있습니다. 

{
    "name": "three_webpack",
    "version": "1.0.0",
    "description": "",
    "main": "index.js",
    "scripts": {
      "build": "webpack"
    },
    "author": "",
    "license": "ISC",
    "devDependencies": {
      "webpack": "^5.74.0",
      "webpack-cli": "^4.10.0"
    }
}

npm run build를 하기 전에 다음과 같이 셋팅을 해줘야 작동을 한다.

4.1 CSS 셋팅하기

npm i -D css-loader

CSS를 사용하기 위해서는 css-loader를 설치해야 합니다.

module: {
    rules: [
        {
            test: /\.css$/,
            use: ["css-loader"],
        },
    ],
},

webpack.config.js 파일에 다음을 추가합니다. 그럼 작동이 됩니다.

npm i -D style-loader

하지만 브라우저에는 아직 나오지 않습니다. 그래서 style-loader를 설치하여 화면에 표시할 것입니다.

module: {
    rules: [
        {
            test: /\.css$/,
            use: ["style-loader", "css-loader"],
        },
    ],
},

style-loader를 추가하면 브라우저에 적용되는 것을 확인할 수 있습니다.

npm i -D file-loader

파일을 추가하기 위해서 file-loader를 설치합니다.

module: {
    rules: [
        {
            test: /\.css$/,
            use: ["style-loader", "css-loader"],
        },
        {
            test: /\.svg$/,
            use: ["file-loader"],
        },
    ],
},

동일하게 모듈에 추가하면 이미지도 추가됨을 확인 할 수 있습니다.

import Img from "./webpack.svg";

export function view() {
    document.querySelector("#app").innerHTML = `
        <h1>hello world</h1>
        <img src = "${Img}" alt="img" />
    `;
}

main.js에도 이미지 파일을 추가하면 화면에 나오는 것을 확인 할 수 있습니다.

 module: {
    rules: [
        {
            test: /\.css$/,
            use: ["style-loader", "css-loader"],
        },
        {
            test: /\.svg$/,
            use: [
                {
                    loader: "file-loader",
                    options: {
                        publicPath: "../dist",
                    },
                },
            ],
        },
    ],
},

이미지가 안나올 경우 이미지 경로를 옵셥으로 설정하면 잘 나옵니다.
